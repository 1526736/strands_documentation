Data stored

Each run of the patroller stores log information inside the datacentre, inside a collection named "episodes" within the "autonomous_patrolling" database. Each document within this collection represents an 'event', and has the following fields:

{
    "_id" : ObjectId("524c0f989f9d51724c9d9cd8"),
    "battery level" : 100,
    "mileage" : 0,
    "event_type" : "episode start",
    "stamp" : {
        "secs" : 1380716440,
        "nasecs" : 385165929
    },
    "episode_name" : "1380716440.385143041"
}

The events can currently recorded are: episode start episode finish start waypoint success waypoint failed waypoint bumper pressed bump recovered navigation recovery charging started charging finished * stuck in carpet

Some events store additional fields, specifically episode start stores the waypoint set name, start waypoint stores the waypoint name and bumper recovered stores the number of motor resets

Access to the logs

Directly

The log information can be directly accessed using a MongoDB client library. For example, to get a list of episode names in python:

In [1]: import pymongo
In [2]: client = pymongo.MongoClient("localhost",62345)
In [3]: client.autonomous_patrolling.episodes.find().distinct("episode_name")
Out[3]: 
[u'1380716440.385143041',
 u'1381417368.657524108',
 u'1381418898.806413888',
 u'1381420037.938729047',
 u'1381423217.295277118',
 u'1381424446.833275079',
 u'1381424611.19880056',
 u'1381424677.752501964']

Python library

There is a python library for accessing log information. This lives inside the waypoint_patroller package, and can be used as follows:

In [10]: from  waypoint_patroller import log_util
In [11]: gen = log_util.StatGenerator("localhost", 62345)
In [12]: gen.get_episode_names()
Out[12]: 
[u'1380716440.385143041',
 u'1381417368.657524108',
 u'1381418898.806413888',
 u'1381420037.938729047',
 u'1381423217.295277118',
 u'1381424446.833275079',
 u'1381424611.19880056',
 u'1381424677.752501964']

In [13]: gen.get_latest_run_name()
Out[13]: u'1381424677.752501964'

In [14]: run = gen.get_episode(gen.get_latest_run_name())
In [15]: print run
Episode Name: 1381424677.752501964
Date: 2013-10-10 18:04:37.752527
Run duration: 15:33:07.112034
Distance covered: 19523.671875
Bumper recoveries: 6
Charge cycles: 0

In [17]: run.stamped_mileage[0:3] # list of timestamps and distance covered
Out[17]: 
[[0.0, datetime.timedelta(0)],
 [0.5546875, datetime.timedelta(0, 1, 5573)],
 [9.71875, datetime.timedelta(0, 28, 52328)]]
In [18]: battery = run.stamped_battery # list of timestamps and battery level
In [19]: charges = run.stamped_battery # list of timestamps that the robot went to charging station

Through ROS

The python library has been wrapped as a ROS node: rosrun waypoint_patroller log_services.py This node provides the following services: /ap_log_server/get_episode_names - returns a list of episode names /ap_log_server/get_latest_episode_name - returns the name of the most recent run /ap_log_server/get_episode_summary - takes the episode name and returns summary statistics as JSON /ap_log_server/get_episode_complete - takes the episode name and returns the complete statistics as JSON
